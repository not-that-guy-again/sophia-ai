# Sophia Roadmap

Planned enhancements, ideas, and future work. Items here are not commitments. They are candidates for future development, roughly ordered by priority within each section.

## Consequence Engine Enhancements

### Red Team Node in Consequence Trees

**Component:** `sophia/core/consequence.py`

The current consequence tree prompt asks the LLM to generate "one optimistic path and one risk path" for each candidate action. In practice, LLMs tend to underweight adversarial outcomes when brainstorming consequences. The optimistic path is usually well-developed while the risk path stays surface-level.

Add a dedicated red team node to each consequence tree. This node would be generated by a separate prompt (or a distinct section of the existing prompt) that explicitly adopts an adversarial posture: "You are trying to find the worst realistic outcome of this action. Think like someone who wants to exploit this decision." The red team node should explore manipulation vectors, cascading failures, precedent abuse, and second-order effects that a cooperative framing would miss.

This could take several forms:
- A third mandatory root node with a red team system prompt
- A post-processing pass that critiques the existing tree and injects missed branches
- A separate LLM call with a dedicated adversarial persona

The red team node output feeds into the same evaluation panel as other consequence nodes. No changes to the evaluators or risk classifier should be needed.

### Transaction-Level Risk Floors

**Component:** `sophia/tools/base.py`, `sophia/core/risk_classifier.py`

Some tools are inherently higher risk regardless of context. Allow individual tools to declare a `risk_floor` that sets a minimum risk tier. For example, a `transfer_funds` tool could declare `risk_floor = "YELLOW"`, meaning it always requires confirmation even if all four evaluators score it positively.

### Consequence Tree Caching

**Component:** `sophia/core/consequence.py`

For repeated or similar actions, consider caching trees keyed by tool name + parameter shape + hat name, with a TTL.

## Evaluator Improvements

### Per-Hat Evaluator Weight Documentation

**Component:** `docs/CREATING_HATS.md`

Add guidance on weight distribution for different risk profiles. Finance hats should likely increase authority weight, content moderation hats might increase tribal weight, internal tooling hats might increase domain weight.

### Multi-Model Evaluator Panel

**Component:** `sophia/core/evaluators/`

Run different LLM models for each evaluator to reduce correlated failure modes.

## Hat System

### Hat Stacking

**Component:** `sophia/hats/registry.py`

Support equipping multiple hats simultaneously with merged tool sets and combined constraints.

### Community Hat Registry

**Component:** new `sophia/hats/marketplace/`

Package hats for distribution via pip or git with a template generator and validation suite.

## Infrastructure

### Persistent Audit Log (Phase 5)

**Component:** `sophia/audit/`

Implement the full audit schema. Already designed but not yet built.

### Outcome Tracking and Feedback Loops

**Component:** `sophia/audit/`, evaluator prompts

Track actual outcomes and compare against predicted consequence trees to tune evaluators over time.

## UI

### Consequence Tree Visualization Improvements

**Component:** `ui/src/components/ConsequenceTree.tsx`

Color gradients for harm_benefit scores, node sizing by probability, and interactive drill-down. Highlight the red team node with a distinct visual treatment.
